## 会议讨论目标与议题

# 主要议题分为五块

1. 初期阶段做脑出血病种聚焦
2. 脑出血从发病到复诊的真实临床闭环细节科普
3. 脑出血闭环AI赋能
4. 初期阶段数据集收集
5. 医院侧协作支持

# 最终目标：
输入病例描述+影像+病史，然后输出完整治疗路径、手术方案、药物方案

# 初期目标：
脑出血病例放射报告生成。 原因：结构化的MLLM放射报告生成最近的NC子刊（2025）上有相关范式可以借鉴， 包含新定义的相关评测指标。 

# 预期交付结果：（初期）
和医生相似的放射报告生成，相近的描述和正确率

# 进阶目标：
脑出血初诊放射报告生成+输出急性治疗/手术路径

## 2. 脑出血闭环流程确认

`发病/症状 -> 急诊分诊 -> 头颅CT/首诊判断 -> 住院/ICU/手术决策 -> 治疗与监测 -> 复查影像 -> 复诊/长期结局`

### 需医生评估的关键问题

#### A. 发病及到院前评估

- 一般而言，从入院触发，（只是知道症状）是直接进入正常急诊后转卒中急诊，还是经过急诊导诊直接进入卒中急诊？（这里可能能够利用模型去做一个卒中导诊，已经有人这么做， 但不在导诊）
- 到院前最关键的病史信息有哪些？

#### B. 急诊及初诊流程

- 首诊时标准流程细节？
- 第一个关键影像是否通常是平扫 CT？（网上共识）什么时候会加 CTA/MRI？
- 在首诊阶段，哪些CT特征是关键的判断项？ （做急诊的ai辅助诊疗， 可用于构建合适结构的提示词微调方法，等价于引入医学先验）

#### C. 住院及治疗管理
- 哪些 CT 特征提示保守治疗/ICU 监测/手术干预？  （做急诊的ai辅助诊疗， 可用于构建合适结构的提示词微调方法，等价于引入医学先验，提示模型关注相关部分细节）
- 临床上最担心哪些并发症和恶化风险？  （医学先验）

#### D. 复查与临床结局评估

- 复查影像的典型时间点是什么？
- 复查CT上最关键的变化指标有哪些？ （如果后续做复诊的ai辅助诊疗， 可用于构建合适结构的提示词微调方法，等价于引入医学先验 ）

## 5. 医生参与协作机制


### 5.1 预期医生协助的内容

1. 帮我们定义第一阶段病种范围和纳入标准，协助高质量医学先验的结构化提示词注入制定。
2. 提供大规模多模态图文对齐数据。
3. 抽样复核数据质量和结构化抽取结果。
4. 对gt报告与predict报告做偏好打分或可用性打分。
5. 对失败案例做专家复盘，提供改进方向，形成闭环。

### 5.2 具体支持内容（初步预期）

#### 5.2.1 诊断报告结构化效果核对

为评估并校正结构化诊断报告时的准确性，按随机方式从处理结果中抽取5%的样本进行人工核对。

#### 5.2.2 大规模多模态图文对齐数据集的标准化采集与采样构建
可借鉴的规模：3D-BrainCT 数据集  train: 7,751名患者，15247个CT（多次扫描）test: 1,938名患者测试，3638个CT  total : 9689名患者，共18885 。  + 对应人工放射报告

#### 5.2.3 图灵测试标注需求

为评估由模型生成的影像报告在临床可辨识性与实用性方面的表现，现存的可借鉴的图灵测试组织方式如下：

-----测试样本 132份报告，同一病例两份报告：一份人类专家，一份为模型生成，11名医师参加评审。

实验分两阶段进行：

第一阶段，评审者仅查看两份报告的文字内容，回答以下问题：

Q1：哪一份是 AI 写的？（可选择“第一份”/“第二份”/“都不是/都不是”）

Q2：你对自己判断的信心有多大？（按李克特量表 1–5 评分）

Q3：影响你判断的主要因素是什么？（如逻辑连贯性、细节具体程度、行文语气、写作质量等，评审者可多选）

第二阶段，系统展示对应病例的CT 影像与两份报告，评审者在复核影像后可修改或确认第一阶段的判断（再次回答 Q1–Q3，并更新信心评分）。

#### 5.2.4 报告评测打分需求
组织临床专家对抽样的GT报告与模型生成的predict报告分别独立评分（每组各100份）。

为了将专家的主观临床经验转化为可计算的客观数据，我们将采用学术界通用的5分制李克特量表。参与评测的专家需针对5个独立维度（诊断准确性、语言质量、放射学细节、信息密度比例、健全性等效）逐一进行标准化赋分，并给出明确锚点与判定示例以确保评分一致性与可复现性：

1分（极差/强烈不认同）：报告存在致命缺陷（如重大病灶漏诊、定性严重错误），完全误导临床决策，毫无可用性。

2分（较差/不认同）：报告存在明显瑕疵或关键信息缺失，需主治医师进行大幅度重写方可使用。

3分（一般/中立）：报告基本合格，无致命诊断错误，但行文或细节描述需中等程度的修改，勉强达到临床基线要求。

4分（较好/认同）：报告准确且描述较为完整，仅需微调个别措辞或补充少量细节，高度符合临床预期。

5分（极好/强烈认同）：报告完美契合人类高级专家的诊断水平与行文习惯，无需任何修改即可作为最终报告直接发布。

### FORTENC + 大模型医学评测（2025NC 评测标准） 部分结构化拆解示例 
FORTE 是一种面向放射学报告生成的结构化语义评测方法。 （注意， 放射学报告生成）
先从真实报告和模型生成报告中分别抽取关键信息，再按固定维度进行对齐比较。
这里的真实报告与模型输出，都会先被拆解为结构化语义项，主要包括四类：

- `程度`：大小、范围、轻重程度
- `部位`：解剖位置
- `特征`：病理或影像学表现
- `印象`：综合诊断结论

#### 示例

**真实报告**

- 右侧基底节区脑内出血，约 28 mL
- 血肿破入右侧脑室
- 轻度中线左移
- 考虑高血压性脑内出血

**模型输出**

- 右侧基底节区脑内出血
- 出血量中等，伴脑室受累
- 可见轻度中线偏移
- 考虑高血压相关脑内出血

#### 结构化对比

| 维度 | 真实报告 | 模型输出 |
| --- | --- | --- |
| 程度 | 28 mL；轻度 | 中等；轻度 |
| 部位 | 右侧基底节区；右侧脑室 | 右侧基底节区；脑室 |
| 特征 | 脑内出血；破入脑室；中线左移 | 脑内出血；脑室受累；中线偏移 |
| 印象 | 高血压性脑内出血 | 高血压相关脑内出血 |
