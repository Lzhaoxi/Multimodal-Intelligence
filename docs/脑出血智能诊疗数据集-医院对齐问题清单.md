# 脑出血智能诊疗数据集建设：医院对齐问题清单

## 0. 这份材料是干什么的

这份材料用于和医院合作医生开技术研讨会，目标不是当场把方案定死，而是尽快把下面几件事对齐：

1. 我们到底要做哪一类“脑出血”数据集，病种边界怎么定。
2. 脑出血从症状到诊断、治疗、复诊、结局的闭环到底长什么样。
3. 在这个流程里，哪些环节最难，哪些环节最值得优先用 AI 辅助。
4. 医生希望数据集最终支撑什么样的模型能力、评测结果和临床协作方式。
5. 医院侧愿意提供什么数据、什么标签、什么打分、什么复核支持。

## 1. 先说结论：这次会议最需要对齐的，不是模型名字，而是 5 个边界

建议优先和医生对齐以下 5 个边界：

1. 病种边界：这次的“脑出血”到底包括哪些疾病。
2. 流程边界：我们是只做急诊-住院段，还是要覆盖复诊和长期结局。
3. 数据边界：哪些数据真能拿到，哪些虽然重要但短期不现实。
4. 任务边界：第一阶段到底是做检测、分型、结构化报告、微调问答，还是做手术/预后支持。
5. 责任边界：哪些输出可以做“AI 辅助提示”，哪些不能作为自动诊疗结论。

## 2. 从论文《Towards a holistic framework for multimodal LLM in 3D brain CT radiology report generation》得到的启发

这篇文章的价值，不是说明“已经有现成答案”，而是提醒我们做脑出血数据集时，至少要注意下面几点：

1. 脑部 CT 的 3D 体数据和文本报告必须做强绑定，不能只留最终报告结论。
2. 仅做报告生成不够，数据集最好同时支持检测、结构化抽取、报告、随访对比等多任务。
3. 评测不能只看文本指标，最好包含医生评价、结构化事实、一致性和外部验证。
4. 真正有价值的数据集，不只是“影像 + 报告”，还应该尽可能纳入病程、治疗决策、复查和结局。
5. 对医生来说，最重要的不是模型会不会说，而是模型有没有帮助判断病情严重程度、辅助决策和减少漏诊误诊。

对我们这次项目的直接启发是：

- 不能只盯着“做一个模型”，要先把脑出血全流程里的关键数据节点和关键判断节点找出来。
- 要优先构建可做多模态微调和可做评测的数据，而不是泛泛收集影像和文本。
- 一定要请医生一起定义“什么叫有用、什么叫可信、什么叫不可接受的错误”。

## 3. 脑出血病种范围：先让医生确认我们到底要做什么

下面是建议在会议上让医生确认的候选范围。重点不是我们自己先拍板，而是让医生给出临床上更合理的范围划分。

### 3.1 候选病种范围

建议提问：

1. 本项目中的“脑出血”，医院侧建议优先覆盖哪些病种？
2. 是否只做自发性脑出血，还是把创伤性出血也纳入？
3. 是否把蛛网膜下腔出血、脑室出血、硬膜下/硬膜外血肿、缺血后出血性转化也纳入？
4. 哪些病种在临床上流程差异太大，不适合放在第一阶段一起做？

建议拿给医生确认的候选分类：

- 自发性脑内出血（ICH）
- 脑室出血（IVH）
- 蛛网膜下腔出血（SAH）
- 硬膜下血肿（SDH）
- 硬膜外血肿（EDH）

- 缺血性卒中后的出血性转化

## 4. 脑出血闭环流程：请医生讲清楚真实临床路径

这部分非常关键。我们需要医生把“从发病到治疗再到复诊”的真实路径讲清楚，而不是只给一个理想化流程图。

### 4.1 希望医生按时间顺序解释的流程

建议请医生按照下面的顺序讲：

1. 发病 / 症状出现
2. 急诊分诊与初筛
3. 影像检查与首诊判断
4. 住院 / ICU / 手术决策
5. 住院期治疗与病情监测
6. 复查影像与疗效判断
7. 出院后复诊与长期结局

### 4.2 建议逐段提问

#### A. 发病与到院前

1. 脑出血患者最常见的首发症状有哪些？
2. 哪些症状提示需要最快做头颅 CT？
3. 到院前最关键的病史信息有哪些？
4. 家属/急救系统通常能提供哪些有用信息？

#### B. 急诊与首诊

1. 脑出血患者到急诊后，标准流程通常是什么？
2. 第一个关键影像一般是平扫 CT 吗？什么时候会加 CTA、MRI？
3. 影像科、急诊科、神经外科、神经内科之间谁先看、谁决策？
4. 首诊时最关键的判断项有哪些？

建议医生补充的关键判断项候选：

- 是否真的存在出血
- 出血类型和部位
- 出血量 / 体积
- 是否破入脑室
- 是否中线移位 / 脑疝风险
- 是否存在脑积水
- 是否需要紧急手术 / 介入 / ICU

#### C. 住院与治疗阶段

1. 脑出血住院治疗时，通常有哪些路径分流？
2. 哪些患者进入保守治疗，哪些进入 ICU，哪些考虑手术？
3. 哪些指标最影响治疗方案选择？
4. 哪些情况最需要重复影像检查？
5. 医生实际最担心的并发症有哪些？

#### D. 复查、复诊与结局

1. 脑出血患者一般会在什么时间点复查影像？
2. 复查时医生最关注哪些变化？
3. 复诊时哪些症状和量表最关键？
4. 什么叫“治愈”或“达到临床预期”？
5. 项目如果要纳入长期结局，医院通常能稳定拿到哪些结局字段？

## 5. 在整个流程里，哪些问题最难？哪些最值得 AI 优先介入？

建议不要一上来问“AI 能做什么”，而是先问医生“最痛的点在哪里”。

### 5.1 建议医生先说难点

建议提问：

1. 从急诊到住院再到复诊，哪些环节最容易出错？
2. 哪些环节最依赖资深医生经验？
3. 哪些环节工作量最大、最重复、最耗时间？
4. 哪些环节最容易出现漏诊、误诊、延迟诊断或延迟干预？

| --- | --- | --- | --- | --- | --- | --- |
| 出血检测 | 头颅 CT 上是否存在出血 |  |  |  |  |  |

| 体积/中线移位/脑积水量化 | 结构化严重程度评估 |  |  |  |  |  |

### 5.3 建议现场追问

1. 哪些任务医生会真正使用，而不是“看起来很厉害但实际不用”？
2. 哪些任务 AI 可以做“提醒/排序/辅助”，但绝不能做自动决策？
3. 哪些任务如果错误一次，医生就会完全失去信任？
4. 如果第一阶段只能做 1 到 2 个任务，医生最希望优先做哪两个？

## 6. 现在主流 AI 辅助脑出血流程大致是什么样

这部分建议你在会上先讲一个很保守的总结，然后让医生确认医院实际情况是否一致。

### 6.1 当前主流 AI 辅助流程的常见形态

当前更常见的 AI 辅助流程，通常不是“AI 直接诊疗”，而是：

1. 在头颅平扫 CT 出图后，做出血检出和危急值筛查。
2. 对疑似出血做分型、定位、出血量/中线移位/脑室受累等结构化提示。
3. 把高危病例前置到工作列表里，加快放射科 / 临床医生处理。
4. 辅助生成结构化报告或病例摘要。
5. 在复查阶段做变化对比或风险提示。

也就是说，现在主体 AI 更像：

- 影像分诊与预警
- 结构化提取与量化
- 工作流排序与报告辅助
- 风险提示与复查对比

而不是：

- 自动给出最终治疗决策
- 自动替代神经外科医生判断

### 6.2 建议向医院确认的现状问题

1. 目前医院有没有在头颅 CT 场景中使用任何 AI 软件或辅助工具？
2. 如果有，主要用于哪一环：筛查、分诊、报告、量化，还是随访？
3. 医生最满意的点是什么？最不满意的点是什么？
4. 医生最不能接受哪类错误：漏诊、假阳性太多、解释不清、报告不稳定，还是临床意义不够？
5. 如果我们做数据集，医院更希望优先支撑“提高检出效率”，还是“支持结构化评估”，还是“支持微调后的多模态问答/辅助报告”？

## 7. 为了构建“多模态微调和评测数据集”，医院必须帮我们确认哪些数据字段

### 7.1 影像数据

建议提问：

1. 第一阶段影像是否只做头颅平扫 CT？
2. CTA 是否稳定可得？是否值得纳入第一阶段？
3. MRI 是否属于后续扩展，而不是第一阶段？
4. 原始 DICOM、序列信息、检查时间能否拿到？
5. 复查 CT 是否可稳定关联到同一患者和同一病程？

### 7.2 文本与病程数据

建议提问：

1. 哪些文本最稳定可拿到？
2. 首诊病历、影像报告、手术记录、病程记录、出院小结、复诊记录是否都能拿？
3. 是否能拿到“最终诊断”而不是只有初步印象？
4. 病程中的关键时间戳是否完整？

### 7.3 结构化标签与量表

建议请医生确认下面哪些字段能稳定给：

- 出血类型
- 出血部位
- 出血量 / 体积
- 是否破入脑室
- 是否中线移位
- 是否脑积水
- 是否存在血肿扩张
- GCS
- NIHSS
- ICH score
- 是否手术 / 手术方式
- ICU / 普通病房去向
- 出院结局
- 90 天 mRS 或其他长期结局

### 7.4 关键问题

1. 哪些字段医生认为“必须标”，否则数据集没有临床价值？
2. 哪些字段医生之间一致性差，需要二次裁决？
3. 哪些字段虽然重要，但医院侧短期拿不到？
4. 哪些标签可以从现有病历中自动抽取，哪些必须人工标？

## 8. 需要医生协助的标注、打分和复核方式

这部分一定要提前说清楚，不然项目会卡在“医生没时间”上。

### 8.1 建议医生协助的方式

1. 帮我们定义第一阶段病种范围和纳入标准。
2. 帮我们定义结构化字段字典和标签规则。
3. 抽样复核数据质量和结构化抽取结果。
4. 对 AI 输出做偏好打分或可用性打分。
5. 对失败案例做专家复盘，告诉我们为什么错。

### 8.2 建议会中直接确认的协作方式

建议当场问清楚：

1. 医生能否接受每周固定时间做小批量标注/复核？
2. 医生更愿意做哪一种协助：
   - 二分类/多分类打标
   - 结构化字段纠错
   - 多个模型输出排序打分
   - 报告是否可用的主观评分
   - 失败案例复盘
3. 是否需要设立“主治/主任最终裁决”机制？
4. 医生能够接受的单次标注/复核工作量是多少？

### 8.3 建议医生打分模板

| 维度 | 1分 | 3分 | 5分 | 医生评分 |
| --- | --- | --- | --- | --- |
| 临床价值 | 几乎无用 | 有帮助但非刚需 | 高价值、真能改善流程 |  |
| 数据可得性 | 很难拿到 | 部分可拿 | 稳定可拿 |  |
| 标签一致性 | 医生差异大 | 需裁决 | 一致性高 |  |
| 模型可行性 | 很难做 | 可尝试 | 很适合第一阶段 |  |
| 上线可接受度 | 医生难接受 | 需大量解释 | 较容易接受 |  |

## 9. 这次会议希望医院最终给我们的输出

建议你最后把会议目标收敛成这些具体输出：

1. 第一阶段纳入的脑出血病种清单。
2. 第一阶段临床闭环流程图。
3. 第一阶段优先做的 1 到 2 个 AI 任务。
4. 第一阶段必需数据字段清单。
5. 第一阶段医生愿意承担的打标/打分/复核方式。
6. 第一阶段可接受的评测目标和错误边界。

## 10. 建议你在会上最后追问的 5 个关键问题

1. 如果第一阶段只能做一个任务，医生最希望 AI 先辅助哪一环？
2. 如果模型漏掉一例高危脑出血，医生最担心的后果是什么？
3. 什么样的模型结果，医生会愿意真正纳入工作流？
4. 哪些字段如果拿不到，项目就不值得做？
5. 医生愿不愿意参与失败案例复盘和主观打分？

- 我们到底做哪一类脑出血
