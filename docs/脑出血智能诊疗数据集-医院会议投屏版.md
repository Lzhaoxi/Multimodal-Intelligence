# 脑出血智能诊疗数据集建设：会议讨论提纲

## 会议希望交流的重点

这次交流，主要希望向临床老师请教并对齐以下内容：

1. 第一阶段到底做哪类脑出血。
2. 脑出血从发病到复诊的真实临床闭环是什么。
3. 在这个闭环中，哪些环节最值得优先用 AI 辅助。
4. 第一阶段数据集最少要收哪些数据、哪些标签。
5. 医院侧适合提供什么样的数据、复核和打分支持。

`发病/症状 -> 急诊分诊 -> 头颅CT/首诊判断 -> 住院/ICU/手术决策 -> 治疗与监测 -> 复查影像 -> 复诊/长期结局`

## 5. 我们想做的主线模型能力是什么

### 当前想法

我们目前更偏向：

- 围绕脑出血构建多模态微调数据集和评测数据集
- 影像 + 文本 + 结构化病程信息共同服务于微调和评测
- 第一阶段不追求“全自动诊疗”，而是先做可用的 AI 辅助能力

### 想请医生重点帮我们判断

- 医生是否认可“第一阶段先做辅助，不做自动诊疗结论”？
- 医生最希望模型最终支撑哪些能力？
- 医生最不能接受哪类错误？

可供讨论的目标能力：

- 检出与分型辅助
- 严重程度结构化评估
- 结构化报告与摘要
- 复查变化对比
- 风险提示与病例排序
- 微调后的多模态问答 / 辅助解释

## 6. 现在主流 AI 辅助脑出血流程大概做到哪一步

### 我们当前的理解

现在主流 AI 辅助流程：

1. 头颅 CT 出图后的出血筛查与危急值提示
2. 出血类型、部位、严重程度的结构化提示
3. 高危病例排序和报告辅助
4. 复查变化对比和风险提示

而不是直接自动替代医生给出最终治疗决策。

- 最满意的点是什么？最不满意的点是什么？
- 如果我们做数据集，医院更希望优先支撑哪一类 AI 能力？
